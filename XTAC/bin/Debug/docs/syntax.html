General Notes

Right now, the language is very primitive.
&& and || operators aren't even supported.
The only comparison operators that work are ==, !=, <, >.
There is no looping construct.
Variable are single bytes (0-255).

Amazingly, this seems to be enough to get a game working.

object attributes:

The following attributes can be tested or set in code:
 holder
 intialdesc - generally you don't need to mess with this.
 description
 n,s,e,w,ne,se,nw,sw,up,down,in,out - You might change these if your game layout changes at run time.
 mass

object properties:

The following properties can be set to 0 or 1:
 portable
 openable
 open
 container - can you put things in it?
 supporter - can you put things on it?
 lit
 on
 wearable
 beingworn
 lockable
 locked
 scenery - is this item explicitly listed

built-in functions:


print("some text"); //prints text
println("some text"); //prints text followed by a new line
add(somevar,5); //adds 5 to somevar
sub(somevar,5); //subtracts 5 from somevar
look(); print room description
move(); move the player
random(somenumber); creates a number between 0 and somenumber

Examples

if (lamp.lit == 1)
{
  lamp.description = "THE LAMP IS GLOWING.";
}
else
{
  lamp.description = "THE LAMP IS OFF.";
}
When refering to an object in code, it is fine if the name has spaces. If you have two objects with the same name, you can refer to them by the id #.
The following will work just fine.

5.description = "YOU NOTICE NOTHING UNUSUAL.";
