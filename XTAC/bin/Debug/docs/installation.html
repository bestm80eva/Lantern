Overview

Latern is an IDE which lets you create a text adventure that can be exported to several classic 8-bit computers.

Platforms currently supported are: Apple 2, Coco 2/3, Commodore 64/128, TRS-80 1/3/4, ZX Spectrum

The IDE writes assembly code for a specific target computer. Other free, 3rd party tools will be needed to assemble the code and attach it to a disk or tape image. A list of these tools is provided below.

Features:
A GUI for creating the game.
A decent, four-word parser
A simple 'C' like language for custom logic.
Easy export to the target platform.

Limitations
No saving/restoring game (yet)
Game Size - Unlike Infocom's Z-Machine, this tool does not support paging. Consequently, the maximum game size is limited to the amount of RAM available in the target machine.

System Requirements

A Windows PC or Linux/WINE.
The .Net runtine environment.

Installation

Download the file release.zip from http://github.com/evancwright/lantern
Unzip the archive in a convenient location. Next, you'll need to get the tools to support building on your intended target platform.

3rd Party Tools

As mentioned before, Latern writes assembly code for several classic computers. However, free 3rd party tools are needed for assembling, creating disk images, and attaching disk images.
Tools used by Latern need to be placed in the bin folder under the Latern installation so the build scripts can find them.

Apple 2
Tools required: java runtime,tasm.exe (Telemark Assembler),AppleCommander.

Coco 2/3
Tools required: lwasm,writecocofile
These are part of lwtools and can be built from source.

Commodore 64/128
Tools required: tasm.exe (Telemark Assembler), C1541.exe.

C1541 is a part of the WinVice C64 emulator suite. If you download WinVice you can also run your finished game using x64.exe

TRS-80
Tools required: TRSTools and Z80asm.exe. These can be downloaded freely from Matthew Reed's site.

ZX Spectrum
Tools required: Z80asm.exe, taputils.exe.


Again, these tools need to go in the bin directory under Lantern.
